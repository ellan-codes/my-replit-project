Build a Fun, Simple Website for “Prenndy Style Party Planning” (with Cart + Email Booking)

Create a colorful, sparkly, kid-friendly website for a party planning business called “Prenndy Style Party Planning”. The site should be fun, simple, easy to navigate, and work great on mobile.

Business Summary

We help parents plan kids’ parties by providing party planning + catering + setup + decorating + serving + entertainment (depending on package). Pricing ranges $45–$200 per hour depending on party size and package.

Site Structure

Make this a single-page site with smooth scrolling + a sticky top nav (or a small multi-page site if easier). Include:

Nav links: Home, Packages, Catering, Gallery, Reviews, Cart, About, Contact

Add a floating “Cart” button (shows item count) on mobile and desktop.

1) Home / Hero (first screen)

Big title: Prenndy Style Party Planning

Subtitle: “Sparkly parties. Delicious food. Stress-free parents.”

Bright, colorful hero image/illustration (balloons/confetti/sparkles)

Two buttons:

Shop Packages

Request a Quote

Add 3 quick “trust badges” beneath:

“Catering + Setup”

“Kid-friendly entertainment”

“Made for busy parents”

Show 4 clickable package cards. Each card has:

Price range per hour

Included services

“Popular” badge on one package (you choose, likely All-Inclusive)

Clicking opens a simple modal to select options before adding to cart.

Implement add-ons properly:

Entertainment add-on toggle (checkbox):

5) Cart (must be clear and simple)

Cart should show:

Package name

Catering size

Guest count (if provided)

Entertainment (Included / Added $25 / Not selected)

Hourly price range (since pricing is a range)

Quantity should generally be 1 (package booking), but allow removing/editing

Add a “Party Duration” selector in cart or checkout:

Number of hours (1–6)
Since pricing is a range, show an estimate like:

“Estimated total: $X–$Y + add-ons”
Calculate:

min = minHourly * hours + one-time add-ons

max = maxHourly * hours + one-time add-ons

At checkout, collect:

Email

Theme (optional: “Taylor Swift”, “Soccer”, “Mermaid”, etc.)

Notes / special requests

Critical behavior

When user clicks Place Order / Request Booking:

Send an email to the business owner that includes:

Package selected

Hour count

Catering size

Guest count

Entertainment (included/added/none) + $25 if added

Estimated total range

Customer contact info

Party date/time/location

Notes/theme/allergies

Show a fun confirmation page:

“Woohoo! Your party request is in!”

“We’ll email you back soon to confirm details.”

Implementation:

Use Node/Express route or serverless function and an email provider:

EmailJS OR Resend OR SendGrid OR Nodemailer SMTP

Store secrets in environment variables (no hardcoding)

Also add a fallback contact method:

A “Contact Us” form that sends the same email (for people not using cart)
